Media capture

Web technologies can not only be used to consume media content, but also provide growing opportunities for capturing content.

Well-deployed technologies

Cameras and microphones available on a huge number of devices can be exploited from Web browsers via the Media Capture and Streams API.

Specifications in progress

Any media content captured, generated or received as a stream can be recorded via the Media Recorder API.

Media content generated or viewed on a given Web application can be turned itself into a media stream (for recording or transmission) via the Media Capture from DOM elements API. More generally, content produced by any other application (or the whole of the display) can also be turned into such a stream via the Screen Capture API.

The Mediastream Image Capture specification caters or more advanced usage of cameras that offer still-pictures settings (e.g. to control zoom or white-balance).

As depth-sensing cameras come to the market, the Media Capture Depth Stream Extensions allows to capture the additional dimension in information they produce.

Features not covered by ongoing work

Audio output capture
The same way one may wish to capture the current screen display as a video stream, it would be useful to enable capturing the current audio output of the device as an audio stream.
