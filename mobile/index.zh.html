本文档总结了 W3C 开发的各类 Web 应用的技术，以及它们如何更具体地应用于移动环境中。

  • 图形和布局

    用来设计引人注目和流畅的用户界面的特性。

  • 设备适配

    创建响应式布局所需的特性，其中的内容自动调整到不同的屏幕、输入机制和其他设备功能。

  • 表单

    构建丰富的表单以及在文本输入仍然是挑战的设备上优化用户输入所需的特性。

  • 数据存储

    保存状态，导出内容，从系统中的文件和服务集成数据的技术。

  • 媒体

    在移动设备上创建身临其境的音频/视频体验所需的特性。

  • 用户交互

    通过设备特定的交互机制（如触摸、振动、通知）吸引用户的特性，并保证这些交互的可访问性（信息无障碍）。

  • 传感器和用户本地交互

    与移动设备中的传感器交互所需的API，包括从加速计和地理定位传感器到基于距离传感器的通信机制。

  • 网络和通信

    通过与网络的交互将 Web 强大的功能和存储能力赋予原本受限的移动设备。

  • 应用生命周期

    将应用与系统其他部分集成的机制并且提供类似原生应用的体验。

  • 支付与服务

    为应用变现所需的特性, 例如集成易用的应用程序内付费。

  • 性能和调优

    监控和改善在移动设备上高度受限的应用程序性能的机制。

  • 安全和隐私

    保证移动设备上隐私和安全的技术与要考虑的因素，因为移动设备保存着用户一些最隐私和机密的数据。
